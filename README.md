# Cavan - 카라반 렌트 플랫폼

## 개요

Cavan은 사용자가 전국의 다양한 카라반(캠핑카)을 둘러보고, 마음에 드는 카라반을 '좋아요'하거나 예약할 수 있는 카라반 렌트 플랫폼입니다. 호스트는 자신의 카라반을 등록하여 다른 사용자에게 선보일 수 있습니다.

이 프로젝트는 프론트엔드와 백엔드가 분리된 구조로 개발되었습니다. (client와 server로 분리)

## 기술 스택

*   **백엔드 (Server)**
    *   Node.js
    *   Express.js
    *   TypeScript
    *   JWT (JSON Web Token) 기반 인증

*   **프론트엔드 (Client)**
    *   React
    *   Vite
    *   TypeScript
    *   CSS

    **기타**
    * 구글 맵 API
    * Gemini Cli
    * Git/GitHub


## 아키텍처

프로젝트는 하나의 레포지토리 안에서 프론트엔드와 백엔드를 각각 `client`와 `server` 디렉토리로 분리하여 관리합니다. 이를 통해 각 영역의 독립적인 개발 및 관리가 가능합니다.

## 주요 기능

*   **사용자 인증**:
    *   회원가입 및 로그인/로그아웃 기능을 제공합니다.

*   **카라반 목록 조회**:
    *   등록된 모든 카라반의 목록을 확인할 수 있습니다.
    *   **정렬**: '거리순', '좋아요순'으로 카라반 목록을 정렬할 수 있습니다.

*   **'좋아요' 기능**:
    *   사용자는 마음에 드는 카라반에 '좋아요'를 누를 수 있으며, 이 정보는 저장됩니다.
    *   **내가 '좋아요' 한 카라반**: 마이페이지에서 내가 '좋아요'를 누른 카라반 목록만 따로 모아볼 수 있습니다.

*   **호스트 기능 (카라반 관리)**:  !!다수의 버그 발생으로 미완성
    *   **등록**: 호스트 계정은 '+' 버튼을 통해 자신의 카라반을 새로 등록할 수 있습니다.
    *   **조회 및 삭제**: 'My Registered Cavans' 페이지에서 자신이 등록한 카라반 목록을 확인하고, 목록에서 삭제할 수 있습니다.

    **채팅 기능**:          !!'유저간 채팅 기능'을 만들고자 했으나, 버그 발생.
    *   유저간 채팅 기능으로 양 방향 소통이 가능합니다.

## 실행 방법

### 1. 백엔드 서버 실행

```bash
# 1. 서버 디렉토리로 이동
cd server

# 2. 의존성 패키지 설치
npm install

# 3. TypeScript 코드 컴파일
npm run build

# 4. 서버 시작
npm start

# 서버가 http://localhost:3000 에서 실행됩니다.
```

### 2. 프론트엔드 클라이언트 실행

> **사전 준비**: 프론트엔드 `client` 디렉토리 내에 `.env` 파일을 생성하고, 아래와 같이 Google Maps API 키를 추가해야 합니다.
>
> ```
> VITE_GOOGLE_MAPS_API_KEY=여기에_당신의_API_키를_입력하세요
> ```

```bash
# 1. (새 터미널에서) 클라이언트 디렉토리로 이동
cd client

# 2. 의존성 패키지 설치
npm install

# 3. 개발 서버 실행
npm run dev

# 클라이언트가 http://localhost:5173 와 같은 주소에서 실행됩니다.
```


## 느낀점
*   프롬프팅을 통한 '바이브 코딩'만으로는 아직 부족하다고 느낌. 다수의 버그가 발생하고, 단순 프롬프팅만으로 해결하기 어려움을 느낌.
*   제대로 활용하기 위해서는 기본 실력과, 다양한 기술 스택을 활용하는 능력이 중요할 것이라고 생각.
*   비록 바이브 코딩의 한계로(한 가지 버그를 잡고자 4시간 가량 노력했지만 실패) 완성하지 못했지만, 여러 기술 스택을 이용해보는 좋은 계기가 되었다고 생각함.
